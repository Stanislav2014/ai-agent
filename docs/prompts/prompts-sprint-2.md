**Ответ:** нет — это GET по конкретному URL, поисковик не вызовет. Для поиска нужен отдельный tool.

### «добавь web_search на DuckDuckGo и обработчик HTML в MD чтобы меньше токенов»

**Скоуп выбран:**
- DuckDuckGo HTML-endpoint (`html.duckduckgo.com/html/`) через POST + `bs4` парсинг `div.result` / `a.result__a` / `.result__snippet`. Альтернативы (Brave, SerpAPI) отклонены — лишние ключи и оверкилл для MVP.
- HTML→MD внутри `http_get` по `Content-Type`. Стриппер `script/style/nav/...` до `markdownify`, cap 5 KB.
- Deps: `beautifulsoup4`, `markdownify`.

## 7. D-05 SAVE=1 (2026-04-23)

### «лог сохраняется?»

**Ответ:** нет, в stdout печатается и теряется при закрытии терминала.

### «да, но лог в гит не добавляй»

**Скоуп D-05:**
- Makefile `SAVE ?=` + `_LOG_FILE = docs/dialogs/run-$(shell date +%Y%m%d-%H%M%S).log`.
- `tee` в `run`-таргете: дублирует stdout+stderr и в терминал, и в файл.
- `.gitignore`: `docs/dialogs/run-*.log` (но `test*-*.log` остаются в репо).
- README обновлён.

**Конфликт:** ранее в `.gitignore` стоял `*.log` + carve-out `!docs/dialogs/*.log` → run-логи попадали бы в git. Решено: добавил отдельную строку `docs/dialogs/run-*.log` после carve-out — последнее правило выигрывает.
